## Code ❕

---

> **TL;DR** 🔖
>
> - **프로그램**은 **목적을 수행하기 위한 명령어들의 모음**입니다.
> - **명령문**은 **특정 작업을 수행하는 단어, 숫자 및 연산자 모음**을 의미합니다.

<br />

**소스 코드** 혹은 그냥 **코드**라고 불리는 **프로그램**은 컴퓨터에게 수행할 작업을 전달하는 **일종의 특수 명령들입니다.**

- 일반적으로 코드는 **텍스트** 형태이지만, 자바스크립트를 사용하면 **브라우저 내 개발자 콘솔에서 직접 코드를 입력할 수도 있습니다.**
- **유효한 형식과 명령어 조합에 대한 규칙**을 컴퓨터 언어라고도 하며, 때로는 **구문**이라고도 합니다.

<br />

### Statements ❓

컴퓨터 언어에서 **특정 작업을 수행하는 단어, 숫자 및 연산자 모음**을 **명령문(Statements)**이라고합니다.

```js
a = b * 2;
```

- 위 코드에서 `a` 와 `b` 는 **변수**라고 하며, 이는 **데이터를 저장할 수 있는 간단한 저장 공간**과 같습니다.
  - 프로그램에서 변수는 **프로그램에서 사용할 값을 보관**하며, 이를 **상징적인 표시자**로 지정합니다.
- 반대로, `2` 는 변수에 저장되지 않은 **단독으로 있는 값**이기에 **리터럴(Literal)** 값입니다.
- `=` 및 `+` 는 **연산자**입니다.
  - **할당 및 수학적인 계산과 같은 작업**을 변수와 리터럴 값을 활용하여 수행합니다.
- 자바스크립트에서 **대부분의 문장은 `;`(세미콜론)으로 끝납니다.**
- 그렇기에, 위 구문을 해석하면 **`b` 의 현재 값**을 가져오고 이에 **`2` 를 곱해 `a` 라는 변수에 할당한다**는 의미를 갖습니다.
- 이를 통해, 프로그램은 **목적을 수행하기 위한 모든 단계를 설명하는 명령문의 모음**이라고 다시 한 번 정의할 수 있습니다.

<br />

## Expressions ❕

---

> **TL;DR** 🔖
>
> - 표현식은 **연산자와 결합된(혹은 그렇지 않은) 변수 및 값 집합**입니다.
> - 단독으로 사용되는 표현식은 **표현문**이라고도 하며, 대개 **호출 표현식**이에 해당합니다.]
> - 프로그램 실행을 위한 자연어 해석을 **컴파일러와 인터프리터**를 활용하여 진행합니다.
> - **자바스크립트 엔진**은 **최적화 가능한 코드를 컴파일**하면서 **인터프리터** 방식으로 실행합니다.

<br />

- 명령문은 하나 이상의 **표현식(Expressions)**으로 구성됩니다.
  - **표현식**은 **연산자와 결합된(혹은 그렇지 않은) 변수 및 값 집합**과 관련된 내용입니다.
- 위 예시를 다시 활용해봅시다.

```js
a = b * 2;
```

해당 명령문에는 **총 4가지의 표현식**이 있습니다.

- `2` 는 **리터럴 값인 표현식**입니다.
- `b` 는 **현재 값을 가져오는 변수 표현식**입니다.
- `b * 2` 는 **곱셈을 의미하는 산술식**입니다.
- `a = b * 2` 는 **할당 표현식**으로 변수 `b` 가 포함된 산술식을 통한 결과를 할당 함을 의미합니다.

**단독으로 사용되는 일반 표현식**은 다음과 같이 **표현문(expression statement)**이라고도 합니다.

```js
b * 2;
```

- 해당 식의 표현문은 **프로그램 실행에 영향을 미치지 않기에(아무것도 반환하지 않기에) 유용하지 않습니다.**
  - 보다 **일반적인 표현문**은 **그 자체가 함수 호출식인 호출 표현식(call expression)**이 이에 해당합니다.

```js
// 호출 표현식(call expression) 예시

alert(a);
```

<br />

### Executing a Program ❓

- 이러한 명령문은 **자연어**이기에 직접 프로그램에 지시하지 못하고 **프로그램을 실행**해야 합니다.
- 이를 컴퓨터 언어로 해석해주는 유틸리티가 **컴파일러와 인터프리터**입니다.

#### 컴파일러와 인터프리터

- 컴파일러는 **컴파일**이라고 하는 **코드 번역을 미리 실행**하고, **컴파일 된 내용을 바탕으로 명령을 실행**합니다.
- 인터프리터는 **프로그램 실행마다 위에서 한 줄씩 코드를 해석하고 실행**합니다.

#### 자바스크립트는?

- 소스 코드가 실행될 때마다 처리되기에 **인터프리팅된다**고 생각하지만, **자바스크립트 엔진**은 **즉석에서 컴파일한 다음 컴파일된 코드를 즉시 실행**합니다.

## Try It Yourself ❕

> **TL;DR** 🔖
>
> - **입력과 출력**을 테스트하기 위한 명령어로 각각 **`prompt()` 함수**와 **`console.log()` 메서드**가 존재합니다.

- 해당 파트에선 자바스크립트로 작성된 간단한 코드로 **프로그래밍 개념**을 소개합니다.
  - 이를 위해 가장 좋은 방법은 **브라우저 콘솔을 열고 직접 실행해보는 것**입니다.

> 개발자 도구에 대한 내용은 생략하겠습니다.

<br />

### Output ❓

```js
a = 21;

b = a * 2;

console.log(b);
```

대표적인 **출력** 메서드인 `console.log` 에 대해 분석해보겠습니다.

- `console.log()` 는 **콘솔에 결과를 출력하는 메서드**입니다.
- 여기엔 설명이 필요한 **2가지 명령문**이 존재합니다.
  - 우선, `log(b)` 부분은 **함수 호출**이라고 합니다.
    - 이는 `b` 값을 가져와 콘솔에 출력하라고 요청합니다.
  - 두 번째로, `console.` 부분은 함수가 위치한 **객체 참조**입니다.
- 이와 비슷하게 결과를 확인할 수 있는 **`alert()` 문**이 있습니다.

### Input ❓

출력을 알아봤으니 **입력**에 대해서도 알아봅시다.

- 입력의 가장 일반적인 방법은 **`form` 요소 등을 통해 텍스트 상자에서 값을 읽는 것**입니다.
  - 이를 대신하여 간단한 테스트를 위해 **`prompt()` 함수**를 사용할 수 있습니다.

```js
age = prompt("Please tell me your age:");
console.log(age);
```

이렇게 되면 **`"Please tell me your age:"` 메시지가 입력란과 함께 팝업창이 등장**하고, 그렇게 전달된 값은 **`console.log()` 를 통해 콘솔에 출력**됩니다.

## Operators ❕

> **TL;DR** 🔖
>
> - **연산자**는 **변수와 값에 대한 작업을 수행하는 방법**이고 **할당, 산술, 비교, 논리 등**의 연산자가 있습니다.

**연산자(Operators)**는 **변수와 값에 대한 작업을 수행하는 방법**입니다.

- 앞에서 보았던 `=` 와 `*` 가 이에 해당합니다.
  - `*` 는 곱셉을 수행합니다. 이와 비슷한 연산자로 `-`, `/`, `%` 등이 있습니다.
  - `=` 는 **할당**에 사용되고 `=` 를 기준으로 **오른쪽 요소를 계산하여 왼쪽에 할당**합니다.

```js
a = 2;
b = a + 1;
```

- 위 경우에 `2` 를 `a` 에 할당하고 변수 `a` 에 `1` 을 더하여 `b` 에 할당합니다.
- 기술적으로 **연산자**는 아니지만 여기서 각 변수를 **선언**하기 위해 `var` 를 사용합니다.
  - 변수를 사용하기 전에 **`var` 를 통해 선언해야 합니다.**
  - 변수를 한 번 선언하면 이후에 필요한 만큼 사용할 수 있습니다.

```js
var a = 20;
a = a + 1;
a = a * 2;
console.log(a); // 42
```

#### 연산자들

- 할당(Assignment): `=`
- 산술(Math): `+`, `-`, `*`, `/`
- 복합 할당(Compound assignment): `+=`, `-=`, `*=`, `/=`
- 증감(Increment/decrement): `++`, `--`
- 객체 프로퍼티 접근(Object property access): `.` (대안 `[]`)
- 동등(Equality): `==`, `===`, `!=`, `!==`
- 비교(Comparison): `>`, `<`, `>=`, `<=`
- 논리(Logical): `||`, `&&`

## Values & Types ❕

> **TL;DR** 🔖
>
> - 프로그램엔 **값**을 나타내기 위한 다양한 표현으로 **타입**을 사용합니다.
> - 자바스크립트에선 **타입 간 강제 변환**을 진행할 수 있으며, **엄격한 동등 연산자(`===`)**를 지향해야 합니다.

<br />

프로그램에서 **값(value)**을 나타내기 위해 **다양한 표현**을 사용하는데 이를 **타입(type)**이라고 합니다.

- 자바스크립트에는 **원시값(Primitive value)**이라고 불리는 타입들이 존재합니다.
  - 숫자(`number`)
  - 문자열(`string`)
  - 불리언(`boolean`)
- 소스 코드에 직접 작성된 값을 **리터럴**이라고 합니다.
  - 문자열 리터럴은 **큰 따옴표(`""`) 혹은 작은 따옴표(`''`)**로 묶여있습니다.
    - 선호에 따라 다르게 사용합니다.
  - 숫자 및 불리언 리터럴은 **그대로 작성**합니다.

```js
"I am a string";
"I am also a string";

42;

true;
false;
```

이외에도 **배열, 객체, 함수 등**도 존재합니다.

<br />

### Converting Between Types ❓

- 자바스크립트에선 **타입 간 전환**을 위해 **강제 변환(coercing between types)**을 사용할 수 있습니다.

```js
var a = "42";
var b = Number(a); // built-in function
```

`Number` 내장 함수를 통해 문자열에서 숫자로 **명시적 강제 변환**을 진행했습니다.

- 이 경우 `a` 와 `b` 는 동등하지 않습니다.
  - 정확하게 말하자면, **엄격한 동등 연산(`===`)**에서 동등하지 않습니다.
- 그렇기에, 느슨한 동등 비교(`==`)를 통해 이를 참이라고 평가할 수도 있지만, 이는 문제를 야기할 수 있기 때문에 지양해야합니다.

## Code Comments ❕

> **TL;DR** 🔖
>
> - 코드는 **개발자에게도 중요**하며, 이를 위해 **주석**을 활용할 수 있습니다.
> - 주석은 **단행 주석과 다행 주석**이 있습니다.

<br />

코드는 컴파일러뿐만 아니라 **개발자에게도 중요합니다.**

- **프로그램 작성 방식**은 팀원 및 미래의 자신에게도 중요합니다.
- 이를 **변수 혹은 함수 이름 짓기**를 통해 원활하게 할 수도 있습니다.

또 하나의 방법은 **코드 주석(Code Comments)**입니다.

- 이것은 온전히 **개발자가 이해하기 위한 텍스트**이며, 컴파일러와 인터프리터는 **이를 무시합니다.**
- 주석에 대한 의견은 다양하기에 이에 대한 보편적인 규칙은 정할 수 없지만, 일부 지침은 매우 유용하게 활용할 수 있습니다.
  - 주석이 없는 코드는 **차선책**입니다.
  - 주석이 너무 많다면 **잘못된 코드라는 신호일 수 있습니다.**
  - 주석은 무엇을 설명하는 것이 아닌 **이유를 설명하는 것**입니다.

주석은 **단행 주석(single-line comment)과 다행 주석(multiline comment)으로 두 가지 유형**이 있습니다.

```js
// 단행 주석

/* 하지만
   이건
   다행 주석 */
```

- 단행 주석은 **단일 명령문 바로 위 또는 줄 끝에 주석을 추가하는 경우**에 유용합니다.
- 다행 주석은 말 그대로 **여러 행의 주석**에 유용합니다.
  - 다행 주석 내에는 `*/` 외에 모든 문자가 가능합니다.

```js
var a = 42; // 42 is the meaning of life

/* The following value is used because
       it has been shown that it answers
       every question in the universe. */
var a = 42;
```

## Variables ❕
